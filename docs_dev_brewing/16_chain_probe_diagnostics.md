## 7. 关键发现

### 准确率规律
- **模型越大准确率越高**：Qwen2.5-0.5B (63.4%) → 7B (86.5%)，符合 scaling 预期
- **任务难度排序一致**：value_tracking > loop/loop_unrolled > conditional > computing > function_call，所有模型都遵循这个排序
- **value_tracking 在所有模型上都最好**（88-96%），function_call 在所有模型上都最差（44-67%）
- **Qwen3 与 Qwen2.5-Coder 在相近参数量下表现接近**：Qwen3-0.6B (69.1%) vs Qwen2.5-0.5B (63.4%)，Qwen3-1.7B (77.3%) vs Qwen2.5-1.5B (75.3%)，Qwen3 略好

### 全对 / 全错
- **全错样本随模型增大快速下降**：Qwen2.5-0.5B 10.5% → 7B 2.9%
- **function_call 全错率始终最高**：即使 7B 仍有 6.3% 的样本全层都错
- **value_tracking 全错率极低**：7B 仅 0.2%（810 个样本中只有 2 个）

### FPCL（首次正确层）
- **7B loop 的 FPCL 中位数 = 0**：超过一半的样本在第 0 层（embedding 之后）就已经被正确分类，说明 loop 任务的答案信息在 hidden state 中出现极早
- **function_call 的 FPCL 普遍最高**（中位数 2-5），信息出现晚
- **FPCL p75-p90 区间很宽**：说明有一批"难样本"信息出现得很晚（接近最后几层）

### 稳定性（最重要的发现）
- **翻转率极高（65-85%）**，是报告中最突出的信号
- 这意味着大多数样本不是"一旦可读就一直可读"，而是在层间反复跳变
- 即使 7B value_tracking（最好的组合），翻转率仍有 48.4%
- **function_call 翻转率最高**：7B 达 83.6%，3B 达 84.6%
- 这恰好是 **brewing-to-resolution** 结构的核心证据：答案信息在中间层"brewing"（不稳定地出现和消失），直到某个层才"resolve"（稳定下来）
